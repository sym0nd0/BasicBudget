# Security Policy

## Supported versions

Only the **latest release** receives security fixes. Older versions are not supported.

| Version | Supported |
|---|---|
| Latest release | ✅ |
| Older releases | ❌ |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

### Option 1 — GitHub Security Advisories (preferred)

Open a [private security advisory](https://github.com/sym0nd0/BasicBudget/security/advisories/new) on this repository. GitHub keeps the report confidential until a fix is published.

### Option 2 — Email

Send a report to the maintainer via the email address on the [GitHub profile](https://github.com/sym0nd0). Please encrypt the message with the PGP key published there if one is available.

## What to include

To help us triage and reproduce the issue quickly, please provide:

- A description of the vulnerability and its potential impact
- Affected component(s) or file(s)
- Step-by-step reproduction instructions
- Any proof-of-concept code or screenshots (attach to the advisory rather than pasting publicly)

## Response timeline

| Stage | Target |
|---|---|
| Acknowledgement | Within 5 business days |
| Initial assessment | Within 10 business days |
| Fix published | Dependent on severity and complexity |

We will keep you informed of progress throughout the process.

## Responsible disclosure

Please give us a reasonable amount of time to investigate and release a fix before any public disclosure. We will credit reporters in the release notes unless they prefer to remain anonymous.

## What not to do

- Do **not** open a public issue or pull request that discloses the vulnerability before a fix is available.
- Do **not** perform destructive testing (data deletion, denial-of-service) against production instances you do not own.
- Do **not** access or modify data belonging to other users.
